### [git] rebase 순서

- rebase 중 conflict가 났다면 `git status`로 문제 위치 알아내기
- 고치고 `git add . && git status`로 rebase 이어가도 되는지 현황 보기
- 문제없을시 `git rebase --continue`
- rebase 완료 후에 강제로 원격 feature 브랜치에 push 해야할 때 : `git push origin feature/XXXX -f`

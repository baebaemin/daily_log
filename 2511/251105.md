## Today I Learned

### [Performance] Inline Critical CSS가 최적화에 좋은 이유

- 일반적으로 CSS 파일은 외부 파일로 로드된다

  ```html
  <!-- index.html -->
  <!-- styles.css는 html과 별도의 파일이기 때문에 로드 필요 -->
  <link rel="stylesheet" href="/styles.css" />
  ```

  - 이 과정에서 브라우저가 서버에게 index.html을 달라고 GET 요청하고, 이어서 styles.css도 GET 요청을 시도한다 = 브라우저가 2번 요청 증
  - 파일 전송이 완료되면 브라우저는 CSS를 적용해서 화면 렌더링을 시작한다.

  - 반면, Inline CSS는 이미 index.html에 CSS가 포함되어있기 때문에 한 번만 요청하고 바로 렌더링이 시작된다.

  ```html
  <style>
    .header {
      color: blue;
    }
    /* critical CSS만 */
  </style>
  ```

- ⚠️ 주의 ⚠️ Inline Style이랑 헷갈리지 말것... ㅎㅎ
  ```html
  <!-- inline style -->
  <div style="color: blue">Hello</div>
  ```

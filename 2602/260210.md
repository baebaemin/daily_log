## Today I Learned

### [git] worktree

- 배경: 메인 브랜치에 병합되지 않을 `chore/ai-tool` 브랜치가 있다. 여러 IDE에서 코드 리뷰 AI 툴을 사용할 수 있도록 `GEMINI.md`, `.agent/`, `.cursor/` 등의 폴더/파일이 생성되어 있는 브랜치다. 각 IDE마다 다른 rules를 취사선택하고 있기 때문에 당분간 main 브랜치에는 병합하지 않을 예정이다.

- 문제: 작업 중인 feature 브랜치에서 `chore/ai-tool` 브랜치의 일부 rule만 가져와 사용하고 싶은데, feature -> main 병합 시 이 파일들을 매번 삭제하거나, `.gitignore`에 추가하거나, staged되지 않도록 관리해야 하는 번거로움이 있었다.

- 시도: `.gitignore`에 AI 설정 파일들을 추가

```bash
# .gitignore에 추가
echo ".agent/" >> .gitignore
echo "GEMINI.md" >> .gitignore

# 파일 가져오기
git checkout origin/chore/ai-tool -- .agent GEMINI.md
```

- 하지만... `.gitignore` 자체가 수정되는 것도 마음에 들지 않았다.

- 해결: `git worktree`를 사용하여 `chore/ai-tool`을 별도 디렉토리에 체크아웃하고, 필요한 파일만 심볼릭 링크로 연결한다. 요렇게 하면 별도 브랜치 관리 없이 항상 최신 AI 설정을 사용할 수 있고, 작업 중인 브랜치를 전혀 오염시키지 않는다.

```bash
# worktree 추가: chore/ai-tool 브랜치를 ../biz-platform-ai-tool에 체크아웃
git worktree add ../biz-platform-ai-tool chore/ai-tool

# 심볼릭 링크 생성 (현재 작업 디렉토리에서)
ln -s ../biz-platform-ai-tool/.agent .agent
ln -s ../biz-platform-ai-tool/GEMINI.md GEMINI.md

# worktree 목록 확인
git worktree list

# worktree 삭제 (더 이상 필요 없을 때)
git worktree remove ../biz-platform-ai-tool
```

- 심볼릭 링크는 Git이 추적하지 않으므로, 작업 브랜치에서 커밋할 때 자동으로 제외된다.

## Today I Learned

### [React] use hook 안에 use hook 안에 use hook ...

- 리팩토링 중에 이미 사용 중인 useAaaHook을 새로 생성할 useBbbHook 내부에서 재사용해도 될지 갑자기 고민이 생겨서 찾아보았다. 커스텀 훅의 목적은 다른 훅들을 묶어서 재사용 가능한 단위로 만드는 것이라서 훅 안에서 다른 훅을 사용하는 건 일반적인 방식이고, 오히려 권장된다고 한다.

  ```js
  // 고민되었었던 예시 패턴
  import { usePositions } from "@/hooks/positions";

  export default function useOfferTemplate() {
    const { data: options } = usePositions("abc_category");
    // ...
  }
  ```

- 그럼 use hook 안에 use hook 안에 use hook ... 2,3단계 depth도 괜찮은건지 궁금해졌는데, 완전 괜찮다고... 😅 사실 이미 커스텀 useHook 뿐 아니라 useEffect useState 등도 Ndepth에 걸쳐서 사용할 때도 있었다.

- 유일하게 지켜야 할 점들이라면,
  - 컴포넌트나 다른 훅의 최상위에서만 호출하기 (= 조건문 / 반복문 / 일반 함수 안에서 호출하면 안됨)
  - 꼬리 문 뱀처럼 돌고돌아 호출하지 않기 (= 같은 순서로 호출하기)

- 규칙에 어긋나진 않더라도 가독성과 디버깅을 위해서 깊이를 너무 깊게 파지 않는 것은 좋겠다 ...

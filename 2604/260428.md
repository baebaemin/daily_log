## Today I Learned

### [git] gitlink

- 부모 repo가 submodule을 어떤 커밋으로 고정해서 바라볼지 기록하는 값. 일반 파일은 git이 파일 내용을 저장하지만, submodule은 내용을 직접 저장하지 않는다. 대신 부모 repo에는 이런 정보만 저장된다

  ```js
  // common/design-system 디렉터리는 design-system repo의 이 커밋을 checkout해라”라는 포인터
  common/design-system -> f657184ce9955cc85b9dde4c873b710bfb1212ac
  ```

- submodule을 최신 커밋으로 바꾸면 실제로는 부모 repo에서 파일들이 바뀌는게 아니라 gitlink 값만 바뀜
- 이 변경도 부모 repo 입장에서는 변경사항이므로 commit해야 다른 사람들이 같은 submodule 커밋을 받는다

  ```js
    // git index 파일 목록 명령어로 확인
    git ls-files -s common/design-system

    // output
    160000 commit f657184ce9955cc85b9dde4c873b710bfb121212 0 common/design-system
  ```

  - 160000 모드 = submodule/gitlink
  - 뒤의 해시 = 부모 repo에 기록된 submodule 커밋

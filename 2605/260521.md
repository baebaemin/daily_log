### [Git] primary worktree & linked worktree

- primary worktree: clone 직후의 디렉터리 (= `.git`이 실제 디렉터리)
- linked worktree: `git worktree add`로 만든 것 (primary의 .git을 가리키는 파일 포인터)

```bash
ls -la /path/monorepo/.git              # → 디렉터리 (primary)
ls -la .claude/worktrees/specs/.git     # → 파일 (linked)
```

### [husky v9] pre-commit + lint-staged로 prettier 자동 수정

- 현재 진행중인 TFT팀은 6명의 FE개발자가 monorepo에서 각자 개발중이다. 팀원마다 사용하는 AI agent가 다르다보니 결과물의 정합성(포맷, 룰셋 등)을 맞출 필요가 있었고, 공통 룰셋은 harness submodule로 공유하기로 했다. 그 일환으로 커밋 시점에 자동으로 prettier가 돌도록 husky 설정도 함께 도입하기로 했다. 

- v9에서는 `package.json`에 `"prepare": "husky"` 스크립트만 등록해두면 `yarn install` 시점에 `.husky/_` 디렉터리가 자동으로 생성된다. 실제 hook은 `package.json` 안이 아니라 `.husky/<hook-name>` 파일로 떼어내서 작성하면 됨. 다만 husky 자체로는 어떤 파일에 어떤 명령을 돌릴지는 책임지지 않아서, 스테이지된 파일에만 prettier를 돌리는 건 `lint-staged`를 같이 도입해 분리한다.

  // package.json
  "scripts": {
    "prepare": "husky"
  },
  "lint-staged": {
    "*.{ts,tsx,js,jsx,json,scss,md}": "prettier --write"
  }
  ```

  ```bash
  # .husky/pre-commit
  yarn lint-staged
  ```

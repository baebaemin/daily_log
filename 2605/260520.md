## Today I Learned

### [Git] GitLab "Apply suggestion" 기능이 pre-push hook을 우회

- 문제 : Husky pre-push에 format:check/lint/typecheck가 걸려 있는데도 develop에 trailing whitespace와 잘못된 간격이 들어간 코드가 머지됐다.로컬에서는 prepush 게이트를 정상 통과했지만, 어떻게 unformatted 코드가 develop에 들어갔는지 불분명했다.
- 해결 : 문제 커밋 메시지가 `"Apply 1 suggestion(s) to 1 file(s)"`인 점에서 GitLab Web UI의 "Apply suggestion" 기능이었던 것 ... 이 기능은 리뷰어가 인라인 suggestion을 클릭하면 서버 사이드에서 직접 커밋하므로 로컬 `git push`가 일어나지 않는다. 그러므로 Husky pre-push hook이 트리거되지 않음!
- 보완 : 막으려면 GitLab server-side push rules(pre-receive hook) 또는 CI 단계에서 format:check를 추가해야 한다.

### [Git Worktree] sparse-checkout

- 문제 : 작업하던 모노레포 영역 중 specs 폴더만 필요한데 관련해서 생성했던 worktree에 `apps/`, `packages/` 등 monorepo 전체 트리가 사용 범위에 비해 무겁게 통째로 들어있었다. 하지만 워크트리 생성 명령인 `git worktree add <path> <branch>`에는 "특정 폴더만 체크아웃" 옵션이 없다.. (심지어 트래킹 중인 파일은 `.gitignore`로도 못 막음)
- 해결 : `git sparse-checkout`을 활용해 worktree 안에서만 특정 디렉터리만 meterialize할 수 있다. 브랜치 자체는 그대로라 커밋/머지 의미는 변하지 않는다.
- 예시 :

```bash
cd .claude/worktrees/specs
git sparse-checkout init --cone
git sparse-checkout set specs .specify
```

### [Next.js] App Router의 dynamic route segment 이름 제약

- 상황 : `/recruit/[siteNo]`와 `/recruit/[recNo]/details`처럼 같은 부모 경로 아래에 다른 dynamic segment 이름을 두 개 두는 라우팅을 설계하고자 했다. 그러나 Next.js App Router는 같은 부모 경로 아래에서 dynamic segment 이름을 하나만 허용해서 `[siteNo]`와 `[recNo]`를 함께 둘 수 없다.
- 해결 : 단일 dynamic segment `[no]`로 통일하고, 페이지 위치에 따라 의미를 다르게 해석(목록·등록은 siteNo, 상세·제안 현황·추천·이력서는 recNo)하도록 했다. ID는 `site-`, `rec-`, `worker-` 프리픽스로 엔티티 타입을 구분.
  - /recruit/[siteNo] # 모집요건 리스트
  - /recruit/[siteNo]/add # 모집요건 등록
  - /recruit/[recNo]/details # 모집요건 상세
- 참고 링크 : https://nextjs.org/docs/app/api-reference/file-conventions/dynamic-routes

### [Git] submodule 버전 고정은 [gitlink](../2604/260428.md#git-gitlink)가 결정

- design-system을 v1.3.0으로 고정하고 싶은데 `.gitmodules`의 `branch = main` 설정이 자동으로 최신 main을 따라가게 만들지 걱정됐다. 하지만 서브모듈 버전은 superproject(= 서브모듈을 포함하고 있는 부모 저장소)가 기록한 **gitlink(커밋 SHA)** 가 결정한다고 한다.
  - superproject 입장에서 packages/design-system/은 보통 디렉토리가 아니라 하나의 특수한 객체(gitlink, mode 160000) 로 staged 됨
- `branch` 필드는 누군가 명시적으로 `git submodule update --remote`를 실행할 때만 적용되는 선택적 힌트이다. "remote에서 새 버전을 가져오라"고 할 때, 어느 브랜치의 tip을 기준으로 할지를 알려주는 것.
- 일반 clone/update에서는 항상 gitlink가 가리키는 커밋으로 checkout 된다.

### [Design System] Curated Facade와 라이브러리 전체 re-export 고민

- 상황 : ToastMessage 디자인 시스템의 버전을 올리면서 react-toastify 라이브러리를 관리하는 주체를 일원화시켰다. 더이상 superproject repo에서 react-toastify의 버전관리를 하지 않아도 되도록 디자인 시스템에서 ToastContainer를 래핑하여 내보내도록 했다.
- 코드리뷰 중, react-toastify 전체를 re-export하는게 어떻겠냐는 의견이 있었다. 이에 다음과 같은 이유로 re-export 하고싶지 않았는데...
  - 사용처에서 어떻게 사용할지 패턴 관리를 할 수 없음
  - 그러므로 사용 패턴을 제공하는 `useToastMessage` 훅을 사용하면 됨
  - API 표면이 30+ symbol로 확장되므로 라이브러리 교체 시 마이그레이션 비용이 증가함
- 그러나 결국 논의 끝에 호출 측 자유도 확보를 위해 react-toastify 전체를 re-export하기로 했다. 단, 팀원들간 react-toastify 라이브러리를 직접 참조하지 않도록 협의되어야 한다. CLAUDE.md나 AGENTS.md에 표기해두는걸로는 문서 레벨의 가드에 불과하므로 eslint rule도 부가적으로 추가했다.

```js
// .eslintrc.js (or eslint config)
module.exports = {
  rules: {
    "no-restricted-imports": [
      "error",
      {
        paths: [
          {
            name: "react-toastify",
            message: "Use @design-system/toast instead.",
          },
        ],
      },
    ],
  },
};
```

## Today I Learned

### [Next.js] App Router layout으로 multi-step form 상태 유지하기

- add 기능의 등록 과정을 `/add/step1` ~ `/add/step4`로 분할 라우팅하는 중이었다. STEP을 넘나들어도 이전에 입력한 값들이 그대로 살아있어야 하는데, 각 STEP이 별도 URL이다보니 페이지 단위로 폼을 관리하면 라우트 전환 시마다 입력값이 통째로 날아가버린다.

- 처음엔 (이전에 하던 방법대로) zustand나 전역 store에 step별 데이터를 따로 들고 있어야 하나 싶었는데, Next.js App Router에서는 layout이 하위 라우트 전환 시에도 unmount 되지 않는 특성이 있었다. `/add/layout.tsx`에 `FormProvider`와 `useForm()` 인스턴스를 두면 step 페이지가 바뀌어도 동일한 폼 인스턴스를 공유할 수 있음

  ```tsx
  // app/site/[siteNo]/add/layout.tsx
  "use client";
  import { FormProvider, useForm } from "react-hook-form";

  export default function AddLayout({ children }) {
    const methods = useForm({ ... });
    return <FormProvider {...methods}>{children}</FormProvider>;
  }
  ```

- layout = "공통 UI 껍데기"로만 생각했는데, "라우트 전환 시 유지되는 상태 컨테이너"로도 활용할 수 있다는 걸 알게 됐다. multi-step form처럼 URL은 나누되 상태는 공유해야 하는 케이스에서 별도 store 없이 react-hook-form만으로도 깔끔하게 풀린다. 다만 layout이 살아있는 건 어디까지나 같은 layout segment 내에서의 클라이언트 사이드 네비게이션일 때 한정이라, 새로고침이 발생하면 폼 상태는 그대로 날아간다. 

- 참고 링크 : https://nextjs.org/docs/app/api-reference/file-conventions/layout

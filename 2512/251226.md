## Today I Learned

### [JavaScript] if (...) { ... } vs if (...) ...

- 타 Repository에 작성된 컴포넌트를 이관하던 중 아래의 로직을 1줄로 바꾸었다
    ```js
    // 이 로직을
    if (isCertificated) {
      onProfileCheck();
    }

    // 중괄호 생략
    if (isCertificated) onProfileCheck();
    ```

- 시각적으로 간결한 코드가 되었지만 명확성을 해치나 싶어 보통 어떤 방식을 어떤 이유로 선호하는지 알아보았다.

    1. 중괄호 있는 버전 : 코드 블록의 범위가 시각적으로 분명하고, 나중에 해당 if문 조건이 아래로 여러 줄을 추가할 수도 있는 가능성을 고려

    2. 중괄호 없는 버전 : 코드가 짧아지므로 간단해보임

- 팀에서는 ESLint로 항상 중괄호를 사용하는 규칙(curly)을 적용하고있진 않지만 Airbnb, Google 등에서는 중괄호를 사용하도록 권장 중

- early return이나 조건문 네스팅 방지를 위한 guard clause를 1줄로 사용할 수 있다면 생략하고, 그 외의 경우에는 휴먼에러 방지를 위해 중괄호를 함께 사용해야겠다 💭

- 관련 링크 : https://eslint.org/docs/latest/rules/curly

## Today I Learned

### [Web API] 모바일 키패드가 화면을 가리고 올라오는 환경인지 감지

- 상황 : 웹뷰를 감싼 모바일 앱을 개발 중인데, BottomSheet 내 input이 모바일 기기에서 focusing 되는 순간 키패드에 가려버린다. 처음엔 앱 개발자분들께 키패드가 올라오는지 여부를 알 수 있는 브릿지를 요청했으나, 인앱뿐만 아니라 모웹으로도 접근 가능한 기능이었기에 앱 브릿지만으로는 해결할 수 없었다.

- 시도 : 이전 프로젝트에서 모웹인지 판단하는 useIsMobile 유틸을 그대로 긁어와 사용하려고 했다.

  ```js
  import { useMediaQuery } from "react-responsive";
  import { isServiceApp } from "@/helpers/appHelper";

  export default function useIsMobile() {
    return useMediaQuery({ maxWidth: 768 }) || isServiceApp();
  }
  ```

- 수정 : 그런데 너비로 판단하기엔 태블릿 대응이 되지 않는다 ... 그래서 navigator.userAgent로 판단하기로 했다.

```js
import { isServiceApp } from "@/helpers/appHelper";

export default function useIsMobile() {
  const isMobileWeb = navigator.userAgent.match(/iPhone|iPad|iPod|Android/i);
  return isServiceApp() || isMobileWeb;
}
```

- 개선 : 코드리뷰 중 이전에 앱뷰 서비스를 개발하셨던 팀원님으로부터 이 조건도 허술하다는 의견을 주셨다. 생각해보니 현재 테스트 기기인 안드로이드와 iOS만 염두에두고 작성한 조건이었던 것이다 ... Ai의 도움을 받아 아래와 같은 함수로 개선했다. 이름도 단순 '모바일 환경인가'를 떠나 virtual 키패드 사용으로 인해 UI의 위치를 변경시켜야하는지 판별하는 useIsVirtualKeyboard로 바꿨다.

```js
import { isServiceApp } from "@/helpers/appHelper";

/**
 * 모바일 키패드가 화면을 가리고 올라오는 환경인지 감지
 * (mobile, tablet 디바이스에서는 true, 데스크탑 환경에서는 false)
 *
 * @returns {boolean} 모바일 키패드 환경 여부
 */
export default function useIsVirtualKeyboard() {
  // 앱 환경이면 키패드가 올라옴
  if (isServiceApp()) return true;

  // SSR 환경 체크
  if (typeof window === "undefined" || !window.navigator) return false;

  const userAgent = navigator.userAgent.toLowerCase();

  // 데스크탑 OS는 키패드가 올라오지 않음 (Windows, macOS, Linux, ChromeOS 등)
  if (
    userAgent.includes("win") ||
    userAgent.includes("mac") ||
    (userAgent.includes("linux") && userAgent.includes("x11"))
  ) {
    return false;
  }

  // 태블릿은 키패드가 올라옴 (iPad, Playbook, Silk 등)
  const tabletRE = /ipad|playbook|silk/i;
  if (tabletRE.test(userAgent)) {
    return true;
  }

  // Android 태블릿 감지: android는 있지만 mobile이 없는 경우 (갤럭시탭 등)
  const isAndroid = /android/i.test(userAgent);
  const hasAndroidMobile = /android.+mobile/i.test(userAgent);
  if (isAndroid && !hasAndroidMobile) {
    return true;
  }

  // 모바일 디바이스 정규식
  const mobileRE =
    /(android|bb\d+|meego).+mobile|armv7l|avantgo|bada\/|blackberry|blazer|compal|elaine|fennec|hiptop|iemobile|ip(hone|od)|iris|kindle|lge |maemo|midp|mmp|mobile.+firefox|netfront|opera m(ob|in)i|palm( os)?|phone|p(ixi|re)\/|plucker|pocket|psp|redmi|series[46]0|samsungbrowser.*mobile|symbian|treo|up\.(browser|link)|vodafone|wap|windows (ce|phone)|xda|xiino/i;
  // ChromeOS는 물리 키보드가 있는 노트북 형태가 대부분이므로 제외
  const notMobileRE = /CrOS/;

  const isMobileDevice = mobileRE.test(userAgent) && !notMobileRE.test(userAgent);

  return isMobileDevice;
}
```

- 배운 점 : Windows, macOS, iOS, Android 외에도 Linux, ChromeOS, blackberry, kindle, opera, ChromeOS ... 등 엄청나게 많은 종류의 모바일 디바이스를 고려해야함을 다시 깨달았다. 물론 이 로직도 완벽하지는 않겠지만, 적어도 초기의 `/iPhone|iPad|iPod|Android/i` 정규식 사용 버전보다는 훨씬 개선되었다.
